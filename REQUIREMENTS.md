# REQUIREMENTS

本書は、`vitepress-docs-hub` のコードベースから読み取れる要件を整理したものです。  
要件は「機能要件」「非機能要件」「制約」に分けて記載します。

## 1. 目的

- 複数プロジェクトのドキュメントを単一の VitePress サイトで集約・公開する
- 利用者がプロジェクト横断で情報に到達しやすいナビゲーションを提供する
- プロジェクト追加時の設定コストを低減する

## 2. 機能要件

### FR-1: プロジェクト定義の読み込み

- システムは `docs/project-docs/` 配下を走査してプロジェクト定義を読み込む
- `name` はディレクトリ名を使用する
- `label/category/description/icon` は `index.md` frontmatter があれば採用する

### FR-2: プロジェクト定義の検証

- システムは `DocProject` の値を検証する
  - `name`: 英数字・ハイフンのみ
  - `path`: `/` で始まり `/` で終わる
- 不正なエントリは警告し、サイト生成対象から除外する
- frontmatter 未設定はエラーにしない

### FR-3: リポジトリリンク解決

- システムは `.gitmodules` を読み取り、`docs/project-docs/<name>` に対応するリポジトリURLを取得する
- 取得できたURLはナビゲーションの `Repositories` から参照できる

### FR-4: グローバルナビゲーション生成

- システムはナビに `ホーム` を表示する
- システムは `プロジェクト` ドロップダウンを表示する
- システムは `Repositories` ドロップダウンを表示する（URLが取得できた場合）
- `Repositories` は外部リンクとして新規タブで開く

### FR-5: サイドバー生成

- システムは `vitepress-sidebar` を使って、各プロジェクト配下のサイドバーを生成する
- システムはトップページ向けサイドバーを独自生成する
  - `docs/` 直下の `.md`（`index.md` 除く）
  - カテゴリ別プロジェクト一覧

### FR-6: タイトル抽出

- `docs/` 直下ページのサイドバータイトルは次の優先度で決定する
  1. frontmatter `title`
  2. ファイル名（整形済み）

### FR-7: ドキュメントサイトのビルド・配信

- ローカル開発サーバーを起動できること
- 本番ビルドできること
- 本番ビルドのプレビューができること

## 3. 非機能要件

### NFR-1: 保守性

- 設定構築ロジックを責務ごとにモジュール分割する
  - 型定義
  - バリデーション
  - プロジェクトロード
  - ナビ構築
  - サイドバー構築

### NFR-2: 型安全性

- `as any` に依存しない設定統合を行う
- VitePress 公式型 (`DefaultTheme`) を利用する

### NFR-3: 回帰防止

- 重要ロジックをテストで検証する
  - ナビゲーション生成
  - バリデーション挙動
- プロパティベーステストを使い、入力パターンに対する堅牢性を確認する

### NFR-4: 可観測性

- 異常検出時に警告ログを出力し、修正箇所を開発者に示す

## 4. 制約・前提

- ドキュメントルートは `docs/`
- 各プロジェクトの配置ルートは `docs/project-docs/`
- サイトエンジンは VitePress
- 一部プロジェクトは Git submodule で管理される
- プロジェクト URL は `/project-docs/<name>/` 形式

## 5. 既知の運用注意

- `docs/project-docs/` 直下にディレクトリを追加すると自動検出でナビに現れる
- frontmatter 未設定時はディレクトリ名ベースの表示になる
