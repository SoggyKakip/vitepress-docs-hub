# REQUIREMENTS

本書は、`vitepress-docs-hub` のコードベースから読み取れる要件を整理したものです。  
要件は「機能要件」「非機能要件」「制約」に分けて記載します。

## 1. 目的

- 複数プロジェクトのドキュメントを単一の VitePress サイトで集約・公開する
- 利用者がプロジェクト横断で情報に到達しやすいナビゲーションを提供する
- プロジェクト追加時の設定コストを低減する

## 2. 機能要件

### FR-1: プロジェクト定義の読み込み

- システムは `docs/.vitepress/config-data/projects.json` からプロジェクト定義を読み込む
- プロジェクト定義には最低限 `name` と `label` を持つ
- カテゴリは JSON のルートキーから導出する

### FR-2: プロジェクト定義の検証

- システムは `projects.json` の構造を実行時に検証する
- システムは `DocProject` の値を検証する
  - `name`: 英数字・ハイフンのみ
  - `path`: `/` で始まり `/` で終わる
- 不正なエントリは警告し、サイト生成対象から除外する

### FR-3: 未登録プロジェクトの自動検出

- システムは `docs/project-docs/` 直下のディレクトリを走査する
- `projects.json` に未登録のディレクトリを検出した場合:
  - 警告を出す
  - `label=name`, `category=未登録` で自動追加する

### FR-4: グローバルナビゲーション生成

- システムはナビに `ホーム` を表示する
- システムは `プロジェクト` ドロップダウンを表示する
- ドロップダウン内はカテゴリでグルーピングする
- カテゴリ未設定プロジェクトはカテゴリ見出しなしで表示する

### FR-5: サイドバー生成

- システムは `vitepress-sidebar` を使って、各プロジェクト配下のサイドバーを生成する
- システムはトップページ向けサイドバーを独自生成する
  - `docs/` 直下の `.md`（`index.md` 除く）
  - カテゴリ別プロジェクト一覧

### FR-6: タイトル抽出

- `docs/` 直下ページのサイドバータイトルは次の優先度で決定する
  1. frontmatter `title`
  2. ファイル名（整形済み）

### FR-7: ドキュメントサイトのビルド・配信

- ローカル開発サーバーを起動できること
- 本番ビルドできること
- 本番ビルドのプレビューができること

## 3. 非機能要件

### NFR-1: 保守性

- 設定構築ロジックを責務ごとにモジュール分割する
  - 型定義
  - バリデーション
  - プロジェクトロード
  - ナビ構築
  - サイドバー構築

### NFR-2: 型安全性

- `as any` に依存しない設定統合を行う
- VitePress 公式型 (`DefaultTheme`) を利用する

### NFR-3: 回帰防止

- 重要ロジックをテストで検証する
  - ナビゲーション生成
  - バリデーション挙動
- プロパティベーステストを使い、入力パターンに対する堅牢性を確認する

### NFR-4: 可観測性

- 異常検出時に警告ログを出力し、修正箇所を開発者に示す

## 4. 制約・前提

- ドキュメントルートは `docs/`
- 各プロジェクトの配置ルートは `docs/project-docs/`
- サイトエンジンは VitePress
- 一部プロジェクトは Git submodule で管理される
- プロジェクト URL は `/project-docs/<name>/` 形式

## 5. 既知の運用注意

- `docs/project-docs/` 直下にディレクトリを追加すると、未登録でも自動検出でナビに現れる
- 意図しない表示を防ぐため、運用上は `projects.json` 登録を標準手順にする
